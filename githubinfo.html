All About GitHub

Q. What is a version control system?

A. Software that helps track the changes made in code over time.

Q. What is Git?

A. A free and open source software that tracks changes made in any set of files and used to coordinate work among programmers collaboratively

Q. What is GitHub?

A. An internet hosting service for sotware development and version control that has the same features as Git, plus access control, bug tracking, software feature requests, task management, continuous integration, and wikis;

Q. What is the difference between Git and GitHub?

A. Git is a software owned by Linux that focuses on version control and code sharing. Github is a service owned by Microsoft that focuses on centralized code hosting.

Q. Who started GitHub and how was it started?

A. GitHub was orginally developed by Chris Wanstrath, P. J. Hyett, Tom Preston-Werner, and Scott Chacon. It was officially launched in April 2008 after it had been released in beta a few months prior.

Q. What company owns it now?

A. Microsoft

Q. How much does a GitHub account cost?

A. It's $4.00 for a Team account, and $21.00 with an Enterprise account.

Q. What is the octocat?

A. It is GitHub's mascot.

Git/GitHub terms:

repository - A "folder" that contains all a project's files.

commit - An individual change made to a file. Contains records of specific changes.

fork - A personal copy of another user's repository.

push - To send committed changes to a remote repository.

pull requests - Proposed changes to a repository submitted by a user.

workflows - A configurable automated process that can run multiple jobs.

issues - Suggested improvements, tasks, or questions related to the repository.

raw button - Opens a file in raw form and removes all HTML formatting.

blame button - Describes the last modification made to each line of a file, and displays the revision, author, and time.
