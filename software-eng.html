Chapter 7: Software Engineering

Software engineering - a systematic engineering approach to software development

integrated development environment (IDE) - systems that combine tools for developing software into a single integrated package

Association for Computting Machinery(ACM)

  • purpose of ACM - dedicated to advancing the art, science, engineering, and application of information technology, serving both professional and public interests by fostering the open interchange of information and by promoting the highest professional and ehtical standards
  • benefits of ACM membership - provides tools and resources to advance your career and enrich your knowledge with life-long learning experiences
  • student ACM membership - helps students gain exposure to a wid range of skills and perspectivs that they won't find in the classroom

Institute of Electrical and Electronics Engineers (IEEE)

  • purpose of IEEE - to foster technological innovation and excellence for the benefit of humanity
  • benefits of IEEE membership - access to new resources, valuable opportunities, and many discounts that help advance your career
  • student IEEE membership - are able to work with professionals, are made aware of scholarships, research programs, internships, employment opportunities, and career advancement programs

Software Life Cycle: Development phase

  • Requirements analysis - focuses on the tasks that determine the needs or conditions to meet the new or altered product or project
  • Design - process of envisioning and defining software solutions to one or more sets of problems
  • Implementation - process of integrating an application into an organization's workflow
  • Testing - process of analyzing to examining a software product or system to determing whether or not it satisfies established conditions

Modular Implementation - the implementation of interoperable systems or devices that can allow for easy adaptation throughout a specific industry or use case

Unified Modeling Language (UML) - a general-purpose, developmental modeling language in the field of software engineering that is intended to provide a standard way to visualize the design of a system

Use Case - a specific situation in which a product or service could potentially be used

Use Case Diagram - a graphical depiction of a user's possible interactions with a system

Data relationships:

  • one-to-one - a relationship where a record in one table is associated with exactly one record in another table
  • one-to-many - a relationship where one row in a table can be associated with many rows in another table
  • many-to-many - a relationship where more than one record in a table is related to more than one record in another table

Design Pattern - a repeatable solution to a software engineering problem

Software quality assurance (SQA) - the ongoing process that ensures the software product meets and complies with the organization's estalished and standardized quality specifications

Software testing:

  • glass box testing - a type of software testing methodology wherein testers can't create tests cases until the code of the program is written completely
  • black box testing - method of software testing that examines the functionality of an application without peering into its internal structures or workings
  • beta testing - a type of user acceptances testing where the product team gives a nearly finished product to a group of target users to evaluate product performance in the real world
  • alpha testing - a type of acceptance testing where in-house team of developers, designers, and quality assurance staff test a product for all possible flaws in a controlled lab environment

    • Documentation:

  • user documentation - information created to help users understand and use a product
  • system documentation - detailed information about a computer system including its architecture, design, data flow, and programming logic
  • technical documentation - the foundation information about the underlying architecture, materials, and process for interfacing with, or building on top of existing technology

    • Intellectual property law

  • software license - a legal instrument governming the use of redistrubtion of a software
  • software disclaimer - a software developer's formal notice to the user of their software that they are denying any and all liability for any damages caused as a result of using the software

    • Best defenses against software liability claims: apply sound software engineering principles during the software's development, to use a level of care compatible with the software's application, and to produce and maintain records that validate these endeavors.
